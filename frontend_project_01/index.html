Project 1: Static Webpage Design

Foundations of Visual Architecture & Technical Integrity

The Objective

Construct a pixel-perfect static foundation. The frontend is the only part of the system the user sees.

Gatekeeper Rule

Engineering Mindset

IPO Model: Input → Process → Output. Structure before code. Information architecture first.

A11Y Standards

Non-negotiable. The web must be perceivable by all users and assistive technologies.

Guidelines & Quality Gate

📐

Information Architecture

Logical sitemap defined first. Semantic tags describe content explicitly.

🧩

DRY & BEM

Modular styling. No inline styles. One copy is easier to maintain.

⚡

Assets & CLS

AVIF/WebP formats. Explicit width/height to prevent layout shift.

✅

Validation

Pass W3C Validator and Lighthouse Audit. Code does not ship until it passes.

🔒 The Architect’s Mandate: “Build it well, for the frontend is the only part of the system the world will ever see.”

Requirements Checklist

Fight Checklist — every item verified

  • ✔ IA: Logical Sitemap defined first.
  • ✔ HTML: Semantic tags (<header>, <main>, <footer>). One <h1>.
  • ✔ CSS: External files only. DRY principle. No IDs for styling.
  • ✔ LAYOUT: Grid for macro page, Flexbox for micro components.
  • ✔ ASSETS: AVIF/WebP formats with explicit dimensions.
  • ✔ VALIDATION: Pass W3C Validator and Lighthouse Audit.
